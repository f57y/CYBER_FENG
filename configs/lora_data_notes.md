# LoRA 数据整理说明

这两个模板都采用 `jsonl` 格式，一行就是一条训练样本。

如果你打算把整理后的数据公开发布，请先单独确认原始素材来源、引用范围和二次分发边界。

`lora_qa_template.jsonl`

- 适合整理直播问答、粉丝提问和峰哥回答。
- 推荐保留 `messages` 结构，后续做聊天式 SFT 更直接。
- `qa` 里只保留两类内容：念题、答题。
- `user` 写成整理后的真实问题，不要只写关键词；如果原始素材是网友刷屏留言，可以合并整理成一句自然提问。
- `assistant` 写成峰哥的念题加答题，可以先接一下题再进入回答，但不要把念题单独拆成另一轮对话。
- 尽量保留峰哥的整活、损话、调侃和口头劲儿，这些往往正是风格的一部分。
- 只删除明显没有结构的碎片、纯口误、纯重复，以及完全不服务当前问答的现场噪音，比如“看公屏”“下一个”“连麦上来”这类。

`lora_monologue_template.jsonl`

- 适合整理峰哥单口输出、口播、直播观点段落。
- `instruction` 不是直播原题也没关系，可以由你根据内容补成一个自然的问题或主题句。
- `output` 写整理后的峰哥表达，重点保留观点结构、语言风格，以及整活时的攻击性、调侃感和节奏。

整理时建议遵守这几个原则：

- 一条样本只表达一个核心观点，不要一条里塞太多主题。
- 尽量保留峰哥的判断和表达习惯，不要改写成通用鸡汤，也不要为了“干净”把味道洗掉。
- 可以适度整理语序和压缩重复，但不要把整活、贫嘴全抹平。
- 去掉明显依赖现场语境、且离开直播场景就完全看不懂的内容，比如纯喊人、纯对公屏互动、纯接上下文的碎句。
- 如果一段内容质量一般，宁可不要，也别为了凑量硬加。
